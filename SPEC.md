# VITA — ფიჩერების სპეციფიკაცია (რაუნდი 2)

ანალიზი 8 ახალი მოთხოვნისა. თითოეულზე: **მიზანი**, **prototype scope** (რას ვაკეთებთ ახლა,
სუფთა client-side), **ინტეგრაციაზე დამოკიდებული ნაწილი** (რა საჭიროებს რეალურ API-ს / გასაღებს),
**მონაცემები/ეკრანები**. ბოლოს — აგების რიგი.

> პრინციპი: ყველაფერი ჯერ აიგება prototype-რეჟიმში (მუშაობს გასაღების/API-ს გარეშე),
> რეალური ინტეგრაციებისთვის დარჩება სუფთა „ნაკერი" (seam), რომ მერე ჩაერთოს.

---

## 1. ვარჯიში — კინეტიკური (კამერით) თრექინგი
**მიზანი:** კამერა ჩართო, აპმა ნახოს და დაითვალოს რეპები — დაადასტუროს, რომ ნამდვილად გააკეთე.

- **Prototype scope:** „კამერით ვარჯიში" რეჟიმი ვარჯიშის ბარათზე → ხსნის კამერას (`getUserMedia`),
  on-device **pose tracking**-ით (MediaPipe Tasks — Pose Landmarker, CDN, უფასო, ლოკალური) ითვლის
  რეპებს არჩეული სავარჯიშოსთვის (ჩაჯდომა/აზიდვა — მუხლის/იდაყვის კუთხის ციკლით). ცოცხალი მთვლელი,
  ხმოვანი/ვიზუალური დათვლა, ბოლოს „შესრულდა N რეპი" → ვარჯიში ავტომატურად მოინიშნება.
- **Fallback:** კამერის/მოდელის გარეშე — მანუალური მთვლელი (+/− ღილაკები) + ტაიმერი.
- **მონაცემები:** `state.workoutLog` (რეპები/თარიღი). **ეკრანი:** workout day → „კამერით დაწყება".
- **შენიშვნა:** რეალური სიზუსტე დამოკიდებულია მოწყობილობაზე/განათებაზე; ეს on-device-ია, კონფიდენციალური (ვიდეო არსად იგზავნება).

## 2. ვიზიტის დაჯავშნის სრული ფლოუ + კლინიკები
**მიზანი:** დაჯავშნა → კალენდარში → ვიზიტის შემდეგ დადასტურება („ვიყავი") → ვიზიტის შედეგების ატვირთვა.
კლინიკები: თუ ინტეგრირებულია — დაჯავშნა სიიდან, საუკეთესო ვარიანტებით (ფასი/ლოკაცია); თუ არა — ნახვა + ტელეფონი + მანუალური დადასტურება.

- **Prototype scope:**
  - **კლინიკების დირექტორია** თითო შემოწმებისთვის: სახელი, მისამართი/მანძილი, ფასი, რეიტინგი, ტელეფონი;
    სორტირება **ფასით / ლოკაციით / რეიტინგით** („best option").
  - **დაჯავშნის ფლოუ:** აირჩიე კლინიკა → თარიღი/დრო → დაადასტურე. სტატუსი: `დაგეგმილი → შემდგარი`.
  - **ვიზიტის შემდეგ:** „იყავი ვიზიტზე?" → დადასტურება → „ატვირთე შედეგები" (გადადის Results-ზე იმ პანელით).
  - **2 რეჟიმი:** `integrated` (დაჯავშნა აპშივე — ღილაკი „დაჯავშნა") და `manual` (ტელეფონი + „დავჯავშნე ხელით").
- **ინტეგრაციაზე დამოკიდებული:** რეალური კლინიკების API/ფასები/თავისუფალი დროები. ახლა — demo დირექტორია (რეალისტური მონაცემებით), seam მზად.
- **მონაცემები:** `state.bookings = [{checkupId, clinicId, datetime, status}]`. **ეკრანები:** clinic finder, booking, visit detail.

## 3. კალენდრის ვიზუალიზაცია + Google/ტელეფონის კალენდართან მიბმა
**მიზანი:** ვიზუალური კალენდარი + სინქი Google Calendar-თან ან ტელეფონის კალენდართან.

- **Prototype scope:** **თვიური ხედი** (grid) — დაჯავშნილი ვიზიტები, მედიკამენტები, ჩექაფები წერტილებით;
  დღეზე დაჭერით — დღის დეტალები. „დაამატე Google Calendar-ში" (web-intent ბმული) + **.ics** (უკვე გვაქვს, ტელეფონის/Apple/Google კალენდარი).
- **ინტეგრაციაზე დამოკიდებული:** ცოცხალი ორმხრივი Google Calendar სინქი → OAuth (Google Calendar API). ახლა — add-to-calendar ბმული + .ics (ცალმხრივი). Seam მზად OAuth-ისთვის.
- **მონაცემები:** იყენებს bookings/meds/checkups. **ეკრანი:** Calendar (თვიური).

## 4. სქესისთვის დამახასიათებელი ჯანმრთელობა
**მიზანი:** სქესის მიხედვით სპეციფიკური ჯანმრთელობის ბლოკები.

- **Prototype scope:** უკვე გვაქვს sex-aware **სკრინინგები** (პროსტატა/მამოგრაფია/საშვილოსნოს ყელი…). გავაფართოვებთ:
  კაცი — ტესტოსტერონი/პროსტატა fokus; ქალი — ჰორმონალური ციკლი, რკინა/ფერიტინი, ძვლის სიმკვრივე.
  პროფილსა და care plan-ში სქეს-სპეციფიკური სექცია/რჩევები.
- **მონაცემები:** `profile.sex` (არსებობს). **ეკრანი:** Profile/Care plan სექციები.

## 5. ქალის ციკლის (დღეების) ჯანმრთელობა
**მიზანი:** მენსტრუალური ციკლის თრექინგი (მხოლოდ ქალებისთვის).

- **Prototype scope:** **ციკლის თრექერი** — ბოლო მენსტრუაციის თარიღი + ციკლის სიგრძე → ფაზის გამოთვლა
  (მენსტრუაცია / ფოლიკულური / ოვულაცია / ლუტეინური), შემდეგი თვის პროგნოზი, სიმპტომების ჩანიშვნა,
  ვიზუალური **ციკლის რგოლი**. ჩანს მხოლოდ `sex === woman`-ისთვის (მენიუში/პროფილში).
- **მონაცემები:** `state.cycle = {lastPeriod, cycleLen, periodLen, logs:[]}`. **ეკრანი:** Cycle.

## 6. წყლის დალევის ვიზუალიზაცია
**მიზანი:** წყლის ჩაწერა + ვიზუალი.

- **Prototype scope:** **წყლის თრექერი** — შესავსები ბოთლი/ჭიქის ანიმაცია, tap-ით +1 ჭიქა (250მლ),
  დღიური მიზანი 2–2.5ლ, პროგრესი, დღის გადატანა. ჩაერთვება დღის გეგმაში/dashboard-ზე ვიჯეტად.
- **მონაცემები:** `state.water = { '<date>': ml }`. **ეკრანი:** ვიჯეტი Plan/Home-ზე + ცალკე tap-tracker.

## 7. „Elements" / ქულების სისტემა + გადაცვლა
**მიზანი:** კარგი ქცევისთვის (დავალებები/სერია) აგროვებს **Elements** (ბრენდის jelly-ელემენტები) / ქულებს;
გადაცვალოს ქულაში ან **vitaapp.ge subscription**-ში.

- **Prototype scope:** **rewards ძრავა** — თითო შესრულებულ დავალებაზე/მედიკამენტზე/ვარჯიშზე +ქულა; სერია/streak ბონუსი;
  დაგროვებული Elements (ფერადი jelly) ვიზუალურად; **Rewards ეკრანი** — ბალანსი, ისტორია, „გადაცვლა" (ფასდაკლება / subscription კოდი / vitaapp.ge).
- **ინტეგრაციაზე დამოკიდებული:** რეალური გადაცვლა vitaapp.ge-ზე → მათი billing/promo API. ახლა — ქულის დაგროვება + „გადაცვლის" demo (კოდი/ბმული vitaapp.ge-ზე).
- **მონაცემები:** `state.points`, `state.elements`, `state.rewardLog`. **ეკრანი:** Rewards.

## 8. vitaapp.ge ინტეგრაცია (რეგისტრაცია/ინფო)
**მიზანი:** ვიტა აპის საიტიდან ინფო/რეგისტრაცია.

- **Prototype scope:** ბმულები vitaapp.ge-ზე (რეგისტრაცია/subscription/ინფო), landing-ზე „დარეგისტრირდი vitaapp.ge-ზე" CTA,
  აპში „გახსენი vitaapp.ge". თუ მოგვცემ რეალურ კონტენტს/ენდპოინტებს — წამოვიღებთ (plan, ფასები).
- **საჭირო შენგან:** vitaapp.ge-ის **რეალური URL/კონტენტი** (რა უნდა გამოჩნდეს, რა ლინკები).
